## <HoverUnderline>What is a Hash Set?</HoverUnderline>

A **hash set** is a data structure that stores unique elements without any particular order. It’s very similar to a hash map, but instead of storing key-value pairs, a hash set only stores keys (or values). The main feature of a hash set is that it automatically removes duplicates, so you don’t have to worry about adding the same item twice.

Internally, a hash set uses a **hashing function** to store elements in an underlying array. This function maps each element to a specific index, which allows for fast insertion, deletion, and search operations, typically in `O(1)` time.

## <HoverUnderline>Key Features of Hash Set</HoverUnderline>

1. **Uniqueness**: All elements in a hash set are unique. If you try to insert a duplicate, it will simply be ignored.
2. **No Order**: Unlike arrays or lists, a hash set does not maintain the order of its elements. It’s not like a sorted collection—elements are stored in random order.
3. **Efficiency**: Inserting, deleting, and searching for an element takes constant time on average, thanks to the hashing function.

## <HoverUnderline>Why Hash Sets are Useful</HoverUnderline>

Hash sets are a great tool when you need to:
1. **Eliminate duplicates**: They automatically ensure that no duplicate elements are stored.
2. **Perform fast lookups**: Checking if an element exists in a set is extremely fast (O(1) on average).
3. **Efficient insertions and deletions**: Adding and removing elements is also very efficient, making hash sets ideal for problems where you need to manage collections of unique items.

Hash sets are widely used in scenarios like:
- Removing duplicates from a list.
- Checking if an element exists in a collection (like in membership tests).
- Storing unique items in large datasets.

In some cases, where order is important, a hash set might not be the best choice, since it doesn't maintain the order of elements. But for problems focused on uniqueness and fast lookups, hash sets are often the go-to data structure.

  <TabsContent value="cpp">
    ```cpp
    #include <iostream>
    #include <unordered_set> // Include the library for hash sets

    int main() {
        // Create a hash set
        std::unordered_set<int> numbers;

        // Inserting elements into the set
        numbers.insert(10);
        numbers.insert(20);
        numbers.insert(30);

        // Attempting to insert a duplicate (this will be ignored)
        numbers.insert(20); // Duplicate, will not be added

        // Checking if an element exists
        if (numbers.find(20) != numbers.end()) {
            std::cout << "20 is in the set." << std::endl;
        }

        // Deleting an element
        numbers.erase(10);

        // Display all elements in the set
        std::cout << "Set elements: ";
        for (int num : numbers) {
            std::cout << num << " ";
        }

        return 0;
    }
    ```

    ```java
    import java.util.HashSet;

    public class Main {
        public static void main(String[] args) {
            // Create a hash set
            HashSet<Integer> numbers = new HashSet<>();

            // Inserting elements into the set
            numbers.add(10);
            numbers.add(20);
            numbers.add(30);

            // Attempting to insert a duplicate (this will be ignored)
            numbers.add(20); // Duplicate, will not be added

            // Checking if an element exists
            if (numbers.contains(20)) {
                System.out.println("20 is in the set.");
            }

            // Deleting an element
            numbers.remove(10);

            // Display all elements in the set
            System.out.println("Set elements: " + numbers);
        }
    }
    ```
